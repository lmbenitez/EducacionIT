Formularios HTML

Son herramientas que permiten una interaccion real (intercambio de datos) entre el cliente y el servidor, es decir, que a diferencia de los demas elementos que existen en una pagina web, la respuesta realizada a la solicitud enviada dependera del contexto en el que se aplique. Es importante aclarar que para crear un formulario debemos utilizar la etiqueta FORM, ya que al enviar los datos al servidor, solamente de tendra en cuenta el contenido de este elemento.

¿Cuantos tipos existen?

Estos se pueden clasificar segun el valor utilizado en el atributo method="..." y el mismo dependera pura y exclusivamente de la prioridad de los datos. GET: se utiliza para las devoluciones por parte del servidor (prioritario), suelen admitir hasta 8kb de datos y estos son accesibles desde la barra de direcciones, por lo tanto se considera un metodo poco seguro. POST: se utiliza para darle prioridad a los datos del usuario (cliente), admiten hasta 8mb de informacion y cifran los datos antes de realizar en envio del mismo, por estos motivos se lo considera un metodo seguro.

¿Como estan compuestos?

En un formulario podemos encontrar dos tipos de elementos, por un lado las etiquetas (LABEL) que sirven de guia para los lectores de pantalla y que el usuario pueda completar o seleccionar los datos correspondientes en el mismo, y por el otro se encuentran los campos (INPUT - TEXTAREA - SELECT) en los cuales el usuario escribira o marcara las opciones relacionadas al elemento anterior. Cabe destacar que la mayoria de los campos en un formulario son elementos INPUT ya que este ultimo admite multiples tipos de datos, que se detallaran mas adelante.

¿Que tipos de datos admite?

Lo datos admitidos por un Control de Formulario se encontraran ligados al atributo type="..." que es exclusivo de la etiqueta INPUT, tambien conocida como Campo de una sola linea . En caso del TEXTAREA este admite cualquier valor, pero se puede limitar la cantidad de caracteres mediante los atributos maxlength y minlength, lo cual es aconsejable si la informacion sera almacenada en una base de datos (Filtrado), este elemento tambien es conocido como Campo Multilinea. Por ultimo, el elemento SELECT permite al usuario marcar una o varias opciones de una Lista desplegable, es importante tener en cuenta que para aquellos campos que no permiten ingresar un valor se debe especificar el atributo value=""

Tablas HTML

Son elementos que permiten organizar la informacion de manera tabular , de ahi su nombre , al igual que en cualquier ambito las mismas se componen de Filas y Columnas, pero a diferencia de la mayoria de los programas que implementan tablas, en el lenguaje HTML las tablas se reprensentan en funcion del Modelo de Cajas . Es decir que partes de una tabla se contemplan como Contenedores y Contenido

¿Como funcionan?

Como se menciona anteriormente, las tablas se crean en funcion del Modelo de Cajas , por lo tanto, dentro de las mismas vamos a encontrar una estructura compuesta por elementos contenedores ( TABLE - TR ) que van a permitir la organizacion de los elementos dentro de la tabla, y por otro lado estan los elementos de contenido (TD) que se utilizan para colocar la informacion que se vera reflejada en la misma. Es importante tener el cuenta que las tablas No se adaptan al tamaño de su contenedor sino que Se adaptan al tamaño de su contenido , rompiendo un poco el esquema de diseño fluido.

¿Poseen Estructura semantica?

Actualmente las tablas poseen Estructura Semantica , es decir que facilitan al navegador la tarea de identificar los diferentes elementos para que luego sean indexados por los motores de busqueda. Se incorporan con HTML5 las etiquetas THEAD para definir la cabecera de la tabla (obligatoria) y por lo tanto su contenido deben celdas de titulo (TH) ; TBODY para colocar los datos de la tabla, aqui se respeta la representacion clasica de elementos en la misma, aunque tambien puede contener titulos en caso de implementar una tabla de Referencias Cruzadas ; por ultimo TFOOT se utiliza para representa un contenido complementario a lo expuesto en la tabla, como podria ser una fila de totales , es opcional y tambien puede contener titulos.
